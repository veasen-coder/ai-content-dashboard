# AI Content Dashboard — Project Guide

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 15 (App Router) |
| Language | TypeScript |
| Styling | Tailwind CSS v4 |
| UI Components | shadcn/ui |
| Icons | lucide-react |
| Package Manager | npm |

## Folder Structure

```
src/
├── app/
│   ├── layout.tsx               # Root layout — sets dark class, TooltipProvider, metadata
│   ├── page.tsx                 # Root redirect → /instagram
│   └── (dashboard)/             # Route group — shares dashboard layout
│       ├── layout.tsx           # Dashboard shell: SidebarProvider + SidebarInset + header
│       ├── instagram/page.tsx   # Instagram Manager section
│       ├── analytics/page.tsx   # Analytics section
│       ├── calendar/page.tsx    # Content Calendar section

│       └── news/page.tsx        # News Consolidator section
├── components/
│   ├── layout/
│   │   └── app-sidebar.tsx      # Sidebar nav component (client component)
│   └── ui/                      # shadcn/ui generated components (do not edit manually)
├── hooks/
│   └── use-mobile.ts            # shadcn/ui hook for mobile detection
└── lib/
    └── utils.ts                 # cn() utility (clsx + tailwind-merge)
```

## Component Conventions

- **Server components by default.** Only add `"use client"` when needed (event handlers, hooks, browser APIs).
- **shadcn/ui components** live in `src/components/ui/` — generated by the CLI, not hand-written.
- **Layout components** live in `src/components/layout/` (e.g. `AppSidebar`).
- **Page files** are the only files in `src/app/(dashboard)/*/page.tsx` — no barrel exports needed.
- Import alias `@/*` maps to `src/*`.

## Dark Theme

The dark theme is applied **globally and permanently** via `className="dark"` on the `<html>` element in `src/app/layout.tsx`. There is no light/dark toggle — the dashboard is always dark.

shadcn/ui's dark-mode CSS variables in `globals.css` handle all component theming automatically.

## Routing

The `(dashboard)` route group wraps all main sections. The group name is invisible in URLs:
- `/instagram` → Instagram Manager
- `/analytics` → Analytics
- `/calendar` → Content Calendar
- `/competitors` → Competitor Tracker
- `/news` → News Consolidator

The root `/` redirects to `/instagram` via `next/navigation` `redirect()`.

## Sidebar

`AppSidebar` (`src/components/layout/app-sidebar.tsx`) uses shadcn/ui's `<Sidebar>` primitives with `collapsible="icon"` mode. Active state is derived from `usePathname()`. The sidebar is wrapped by `SidebarProvider` in the dashboard layout.

## Key Decisions

1. **App Router route group `(dashboard)`** — keeps the shared shell (sidebar + header) DRY without affecting URL paths.
2. **`shadcn` installed locally** (`npm install shadcn --save-dev`) to avoid npm cache permission issues with `npx`.
3. **TooltipProvider at root** — required by shadcn/ui tooltip components; placed in `src/app/layout.tsx`.
4. **No light mode** — dashboard is dark-only; simplifies CSS and avoids hydration flash issues.
5. **Placeholder pages** — all section pages are fully structured with real UI chrome (stat cards, empty states, CTAs) but no live data yet.
